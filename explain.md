# AiStudy Application Documentation

This document provides a comprehensive explanation of the AiStudy platform, detailing its architecture, frontend structure, backend services, and AI generation pipelines.

## 1. System Architecture

AiStudy is a full-stack AI-powered study assistant built with the following technologies:
- **Frontend**: React (Vite), JavaScript, Vanilla CSS, Lucide React (Icons).
- **Backend**: Node.js, Express.js.
- **Database & Auth**: Supabase (PostgreSQL, Row Level Security, Storage Buckets).
- **AI Models**: Groq (Llama-3.3-70b for fast text) and Gemini 1.5 Flash (for structured outputs like quizzes).
- **File Processing**: `pdf-parse` for text extraction, `multer` for handling uploads.

### Communication Flow
1. The user interacts with the React frontend.
2. For database operations (like fetching past notes or saving user details), the frontend connects directly to Supabase using the Supabase JS Client (`src/services/supabaseClient.js`).
3. For heavy AI processing or file parsing, the frontend sends a request to the Express backend (`backend/server.js`).
4. The backend communicates with LLM APIs (Groq/Gemini), processes the data, and returns the result to the frontend.

## 2. Frontend Structure (`/src`)

The frontend is a Single Page Application (SPA) structured around a main `App.jsx` which manages the active view and user session.

### Core Files
- **`App.jsx`**: The entry point that manages routing (Dashboard, Documents, Notes, Quizzes, Doubts, Planner, Flashcards) and sidebar navigation.
- **`main.jsx`**: Bootstraps the React application into the DOM.
- **`index.css` & `App.css`**: Contain the global design system, CSS variables, glassmorphism utilities, and modern animations.

### Views (`/src/views`)
- **`Auth.jsx`**: Handles User Login, Signup, and Password Recovery using Supabase Auth.
- **`Dashboard.jsx`**: The home screen. Displays summary statistics (number of docs, notes, quizzes) and quick actions for recent documents.
- **`Documents.jsx`**: The core upload hub. Features a drag-and-drop PDF uploader. Once uploaded, it triggers the automatic 4-step AI pipeline (Notes -> Quiz -> Flashcards -> Plan) in parallel.
- **`Notes.jsx`**: Displays AI-generated summarized notes. Uses a custom Markdown parser to render clean, readable content.
- **`Quizzes.jsx`**: An interactive testing environment. Users answer multiple-choice questions, and it calculates a score and displays explanations for correct/incorrect answers.
- **`Flashcards.jsx`**: An active recall tool. Renders 3D flipping cards, tracks progress, and allows users to mark cards as "Mastered."
- **`Planner.jsx`**: Displays personalized, day-by-day study plans generated by the AI based on the user's available hours and learning goals.
- **`Doubts.jsx`**: A contextual chat interface. Users can ask questions specifically about an uploaded PDF, or ask general queries.

### Services (`/src/services`)
- **`supabaseClient.js`**: Initializes the Supabase client. Includes a powerful "Mock Mode" that simulates database operations in `localStorage` if environment variables are missing.
- **`aiService.js`**: Acts as an API wrapper for the frontend. It contains functions (`generateNotes`, `generateQuiz`, etc.) that make HTTP requests to the backend. It also contains fallback mock data generators if the backend is unreachable.

## 3. Backend Structure (`/backend`)

The backend is an Express server responsible for securely interacting with AI APIs and parsing files.

### Core Files
- **`server.js`**: The main Express application. Sets up CORS, JSON parsing, and defines the API routes (`/api/extract-pdf`, `/api/generate-notes`, etc.).
- **`supabaseClient.js` (Backend)**: Initializes a secure Supabase client using the `SERVICE_ROLE_KEY` to bypass Row Level Security for automated background tasks.

### Services (`/backend/services`)
- **`aiService.js`**: The core LLM router. It decides whether to use Groq (for fast chat/summaries) or Gemini (for structured JSON). Contains fallback mechanisms if API keys fail.
- **`quizService.js`**: Specialized service for generating structured JSON quizzes.
- **`flashcardsService.js`**: Specialized service for generating Question/Answer pairs for flashcards.
- **`plannerService.js`**: Specialized service for structuring day-by-day study tasks.

## 4. The AI Auto-Generation Pipeline

When a user uploads a PDF in `Documents.jsx`, the following sequence occurs automatically:

1. **Upload**: The PDF is uploaded to the Supabase `study-materials` storage bucket.
2. **Extraction**: The file buffer is sent to the backend (`/api/extract-pdf`), where `pdf-parse` extracts the raw text.
3. **Indexing**: A new record is created in the `documents` table in Supabase.
4. **Parallel Generation**: `Promise.all()` is used to run 4 generation tasks simultaneously:
   - Request to `/api/generate-notes` -> Saves to `notes` table.
   - Request to `/api/generate-quiz` -> Saves to `quizzes` table.
   - Request to `/api/generate-flashcards` -> Saves to `flashcards` table.
   - Request to `/api/generate-plan` -> Saves to `study_plans` table.

## 5. Security & Row Level Security (RLS)

- **JWT Tokens**: All frontend requests to Supabase include the user's JWT token.
- **Policies**: Supabase RLS policies ensure that a user can only `SELECT`, `INSERT`, `UPDATE`, or `DELETE` rows where `user_id = auth.uid()`.
- **Backend Security**: The backend never exposes API keys (Groq/Gemini) to the frontend client.

## Conclusion
The codebase is designed to be highly resilient. If backend APIs fail, or if Supabase keys are missing, the application seamlessly falls back to local storage and mock AI responses, ensuring the UI never breaks during development.
